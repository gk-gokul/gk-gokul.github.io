IntelliAware — Industrial AI Monitoring Platform
Computer Vision · Edge AI · Gemini Multimodal · NVIDIA Jetson Orin

Factory-agnostic process monitoring system deployed on NVIDIA Jetson Orin — a three-tier CV pipeline combining perceptual hashing, LightGlue TensorRT FP16, and Gemini 2.0 Flash multimodal reasoning for real-time step tracking, anomaly detection, and cycle intelligence. No manual tuning: auto-calibrates from reference frames.

  • Three-tier detection pipeline: pHash + histogram EMA (~1ms, CPU) → LightGlue TensorRT FP16 (~20ms, GPU Tensor Cores) → Gemini 2.0 Flash YES/NO (~2s, async, rare fallback)
  • Gemini multimodal classifier: submits before/after transition frame pairs with step descriptions; classifies which manufacturing step just completed — factory-agnostic, no code changes to switch plants
  • Jetson Orin deployment: Dockerized with dustynv/l4t-pytorch base, TensorRT engine auto-export on first run, starts in <5s after initial build; separate docker-compose for dev vs. Jetson
  • MobileNetV3 always-on: runs every 3rd frame for motion detection, camera misalignment alarms, and stuck-process detection — independent of the step classifier
  • PC calibration server: FastAPI backend handles video upload, auto cycle-boundary detection (SSIM + cosine similarity), reference frame extraction, and KO (Knowledge Object) structured storage via MIX store
  • SSE-streaming video monitor endpoint with real-time pacing, job cancellation, cycle log tracking, and bootstrap phase to find start state on mid-process attach

● Deployed on a Real Farm
github.com/gk-gokul
IntelliGrow — RL-Powered Smart Plant Care System
Reinforcement Learning · IoT · ESP32 · Flask · PPO · Real Hardware

A fully functional IoT + AI system for autonomous plant care — inspired by a local farmer and deployed on a real farm. An ESP32 reads live sensor data (temperature, humidity, light, soil moisture) and a PPO reinforcement learning model decides when to water, activating a relay-controlled pump with hardware-calibrated precision.

  • PPO reinforcement learning agent: custom Gym-compatible PlantWateringEnv trained on real sensor logs — reward structure penalizes overwatering (soil >85%) and underwatering (soil <45%), trained over 5,000 timesteps using Stable-Baselines3
  • Hardware-calibrated water delivery: derived flow-rate equation accounting for tube fill delay — Watering Time = (Volume × Flow Factor) + Tube Delay Constant — ensures precise delivery even at small volumes
  • Full IoT pipeline: ESP32 WROOM reads DHT11 (temp/humidity), analog light sensor, and soil moisture sensor over Wi-Fi → HTTP POST to Flask server → RL model inference → relay module + DC motor actuation
  • Real-time Flask dashboard: live sensor charts, latest AI prediction display (WATER / DO NOTHING), manual override button, daily CSV logs — fully working UI in production
  • Anomaly detection for sudden environmental drops; separate motor power supply with capacitor smoothing to prevent ESP32 reboots from voltage spikes during pump activation
PPO (Stable-Baselines3) Gymnasium ESP32 Flask DHT11 / Soil Moisture Relay + DC Motor IoT / Wi-Fi Python Pandas / NumPy
